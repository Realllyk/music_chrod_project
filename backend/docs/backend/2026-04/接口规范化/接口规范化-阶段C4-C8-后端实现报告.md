## 2. C4：audio_sources_controller 改造

### 新增文件
- `backend/pojo/dto/audio_sources_dto.py`
- `backend/pojo/vo/audio_sources_vo.py`

### 接口结果
- `GET /api/audio-sources/list`
  - 返回 `PageVO[AudioSourceVO]`
- `GET /api/audio-sources/oss-files`
  - 返回 `{items, total}`
- `GET /api/audio-sources/<id>`
  - 返回 `AudioSourceVO`
- `DELETE /api/audio-sources/<id>`
  - 返回统一 `Result`
- `POST /api/audio-sources/upload`
  - 返回 `{ok, audio_source_id, audio_name, file_path, format}`

### 实现说明
- 控制器不再直接裸用 `request.args / request.files / request.form`
- 列表与详情统一通过 `AudioSourceVO` 做响应映射
- 删除接口保留统一 `Result`
- 上传接口补充返回 `format`

## 3. C7：files_controller 改造

### 接口结果
- `GET /api/files/oss-url`
  - 返回统一 `Result`

### 实现说明
- 增加 query DTO 校验：
  - `song_id / session_id / audio_source_id` 三者必须且只能传一个
  - `type` 受限于 `audio | melody | chord | recording | source`
- 查询成功后统一包装为 `Result.success({...})`
- 资源不存在或未匹配类型时统一返回 `Result.not_found(...)`

## 4. C8：home_controller + health_controller 改造

### health_controller
- `GET /api/health` → 统一 `Result`
- `GET /api/status` → 统一 `Result`
- `POST /api/db/test` → 统一 `Result`

### home_controller
- 输出文件路由调整为：`GET /api/outputs/<filename>`
- 文件存在时返回 `send_file()`
- 文件不存在时返回 `Result.not_found()`

### 路由兼容性说明
- 旧版路由是 `/outputs/<filename>`
- 本次按迁移计划切到 `/api/outputs/<filename>`，与前端统一 API 前缀保持一致
- 如果线上仍有旧调用方，需同步改前端或在网关层做兼容转发
