# 旋律结果可视化方案｜前端实现报告

## 1. 本次完成内容

### 1.1 修改文件
- `frontend/pages/songs/details.html`
- `frontend/pages/songs/melody.html`

### 1.2 主要实现点

#### A. `details.html` 调整
- 保留原有 `melody_url || melody_path` 判断逻辑
- 旋律结果区域不再展示 MID 原始 URL / path 文本
- 有旋律结果时展示两个按钮：
  - `查看旋律简谱`：跳转 `/songs/melody?song_id=<id>`
  - `下载 MID`：继续下载原始 MID 文件
- 无旋律结果时继续显示 `未生成`

#### B. 新增 `melody.html`
- 从 URL 中读取 `song_id`
- 请求 `GET /api/songs/<song_id>/melody-analysis`
- 当分析接口未返回歌曲标题时，补调用 `GET /api/songs/<song_id>` 获取标题
- 页面结构包含：
  - 返回按钮
  - 歌曲标题
  - Info bar：调 / 拍号 / BPM / 置信度
  - 按小节渲染的简谱卡片区
- 渲染逻辑：
  - 优先按 `result.analysis.notation.measures` 渲染
  - `result.analysis.notation.notes` 用于音符 hover tooltip 和八度信息映射
- 视觉规则：
  - `#1` / `b3` 使用上标展示升降号
  - 使用 `oct1`、`oct2`、`oct3`、`oct-1`、`oct-2`、`oct-3` 类实现上点/下点八度效果
- tooltip 展示字段：
  - `pitch_name`
  - `start`
  - `end`
  - `duration_beats`

#### C. 状态处理
已实现以下状态 UI：
- 加载中
- 空数据（成功但无 `measures` / `notes`）
- 404：
  - melody analysis not found → “该歌曲尚未生成旋律结果”
  - 其他 404 → “歌曲不存在”
- 500 / 其他服务端异常 → 错误提示 + 重试按钮
- 网络异常 / 响应解析异常 → 错误提示 + 重试按钮
- 缺少 `song_id` 参数 → 明确提示，不白屏

## 2. 实现说明

### 2.1 `details.html`
当前旋律结果展示改为按钮式入口，不再暴露底层文件路径，避免页面直接显示技术性路径文本。

### 2.2 `melody.html`
页面采用纯 HTML + CSS + Vanilla JS 实现，样式沿用现有卡片式页面风格。

简谱页面渲染策略：
1. 读取 `song_id`
2. 请求旋律分析接口
3. 解析 `result.analysis.key / time_signature / tempo / notation`
4. 渲染信息栏
5. 按小节输出简谱数字
6. 使用 notes 中的对应音符补充 tooltip 与八度点样式

当接口只返回 notes、不返回 measures 时，页面也提供兜底的连续旋律片段展示。
