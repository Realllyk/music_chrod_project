## 2. 已完成改动

### 2.1 统一切换到 `frontend/js/api.js`

三个页面均已移除原生 `fetch(...).then(x => x.json())` 直连写法，改为统一使用：

- `apiGet`
- `apiDelete`
- `apiPostForm`

并在页面中显式引入：

```html
<script src="/js/api.js"></script>
```

### 2.2 列表接口完成 `r.sources → r.items`

`frontend/pages/audio-sources/index.html` 已完成以下替换：

- `r.sources` → `r.items`
- `r.sources.map(...)` → `items.map(...)`

当前列表页通过：

```js
const r = await apiGet('/api/audio-sources/list', { limit: 100, offset: 0 });
const items = r.items || [];
```

进行渲染。

### 2.3 删除 `r.ok / r.error` 分支，统一改为 try/catch

三个页面中涉及上传、详情加载、删除的逻辑均已改为：

- 成功：直接使用返回结果
- 失败：统一进入 `catch(e)`
- 错误提示统一读取 `e.message`

已清理：

- `if (r.ok)`
- `if (r.error)`
- `if (data.ok)`
- `data.error`

### 2.4 保持 upload/details 字段访问基本不变

按照方案要求，上传页与详情页对后端 VO 字段保持兼容：

- `r.audio_source_id`
- `s.id`
- `s.audio_name`
- `s.format`
- `s.file_size`
- `s.status`
- `s.created_at`
- `s.file_path`

未对这些字段做破坏性改名。

### 2.5 音频预览兼容 `file_url || file_path`

`frontend/pages/audio-sources/details.html` 已改为：

```js
const previewUrl = s.file_url || s.file_path || '';
audioPlayer.src = previewUrl;
```

不再拼接 `API_BASE`，兼容 OSS 直链及后续可能补充的 `file_url` 字段。
