# 接口规范化-阶段C2-前端实现报告

## 1. 实现范围

本次已完成以下页面改造：

- `frontend/pages/capture.html`
- `frontend/pages/recordings/index.html`
- `frontend/pages/recordings/detail.html`

## 2. 已完成改动

### 2.1 capture.html

已完成：

- 接入 `/js/api.js`
- 所有 capture API 调用统一改为 `apiPost / apiPut / apiGet / apiDelete`
- 所有 capture API 调用统一使用 `try/catch`
- 删除 `if (r.ok)` 分支
- 历史列表字段由 `r.sessions` 改为 `r.items`
- `loadRecordings()` 失败时改为显示 `加载失败: ${e.message}`，不再静默吞错
- 删除录音成功后补充刷新历史列表

当前接口对应关系：

- `POST /api/capture/start-recording` → `apiPost`
- `PUT /api/capture/stop-recording` → `apiPut`
- `GET /api/capture/list` → `apiGet`
- `DELETE /api/capture/sessions/:id` → `apiDelete`

说明：

- 本次未额外引入新的 `audio_source_id` 录入 UI。
- 当前 `capture` 页面仍按已批准方案采用“最小改造方案”，继续沿用兼容接口 `/start-recording` 与 `/stop-recording`。
- 现有后端 `start-recording` / `stop-recording` 未要求前端传入 `audio_source_id`，因此本次仅完成接口规范化与错误处理统一；如后续 capture 正式流程切换到阶段 B 同类的“显式选择音源”方案，可再追加 UI 与参数接入。

### 2.2 recordings/index.html

已完成：

- 接入 `/js/api.js`
- 会话列表请求改为 `apiGet('/api/capture/list', { limit: 100 })`
- 列表字段由 `r.sessions` 改为 `r.items`
- 编辑文件名改为 `apiPut('/api/capture/sessions/:id', ...)`
- 删除录音改为 `apiDelete('/api/capture/sessions/:id')`
- 删除 `if (r.ok)` / `r.error` 分支
- 统一使用 `try/catch`
- 编辑成功后仍关闭弹窗并刷新列表
- 删除成功后仍刷新列表

### 2.3 recordings/detail.html

已完成：

- 接入 `/js/api.js`
- 详情请求改为 `apiGet('/api/capture/detail/:id')`
- 删除 `if (r.error)` 分支，改为 `try/catch`
- 保存文件名改为 `try/catch + apiPut`
- 删除录音改为 `try/catch + apiDelete`
- 保持详情字段平铺访问不变
- 保持 `file_path` 用于音频预览，不改为 `file_url`
- 无音频路径时补充隐藏播放器并清理 `src`

### 3.2 API 调用检查

当前 capture/recordings 页面中的 capture API 调用如下：

- `capture.html`
  - `apiDelete('/api/capture/sessions/'+currentSessionId)`
  - `apiPost('/api/capture/start-recording', {})`
  - `apiPut('/api/capture/stop-recording', ...)`
  - `apiGet('/api/capture/list', {limit:20})`
- `recordings/index.html`
  - `apiGet('/api/capture/list', {limit:100})`
  - `apiPut('/api/capture/sessions/'+sessionId, ...)`
  - `apiDelete('/api/capture/sessions/'+sessionId)`
- `recordings/detail.html`
  - `apiGet('/api/capture/detail/'+id)`
  - `apiPut('/api/capture/sessions/'+currentSessionId, ...)`
  - `apiDelete('/api/capture/sessions/'+currentSessionId)`

上述调用均已置于 `try/catch` 中。
