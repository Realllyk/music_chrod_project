# 接口规范化-阶段C3-前端实现报告

## 1. 本次修改文件
- `frontend/pages/artists/index.html`
- `frontend/pages/artists/add.html`
- `frontend/pages/artists/details.html`

## 2. 核心改动

### 2.1 artists/index.html
- 接口调用切换为 `apiGet('/api/artists/list')`
- 列表读取由 `r.artists` 改为 `r.items`
- 本地搜索过滤同步改为基于 `r.items`
- 请求失败时不再静默吞错，改为在列表区域展示 `加载失败：${e.message}`

### 2.2 artists/add.html
- 新增接口调用切换为 `apiPostForm('/api/artists/add', formData)`
- 删除旧的 `if (r.ok)` / `r.error` 分支
- 统一使用 `try/catch`，失败提示使用 `e.message`
- 成功后保持原有清空表单逻辑

### 2.3 artists/details.html
- 详情加载切换为 `apiGet('/api/artists/:id')`
- 保存接口按场景分别切换为：
  - 无文件：`apiPut('/api/artists/:id', data)`
  - 有文件：`apiPutForm('/api/artists/:id', formData)`
- 删除接口切换为 `apiDelete('/api/artists/:id')`
- 加载 / 保存 / 删除全部统一为 `try/catch`
- 页面内移除 `if (r.ok)`、`if (r.error)`、`r.error`
