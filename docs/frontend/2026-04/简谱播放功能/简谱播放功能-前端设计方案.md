# 简谱播放功能 - 前端设计方案

## 1. 需求目标

在 `frontend/pages/songs/melody.html` 新增简谱播放能力：

1. 用户点击播放后，基于后端已返回的简谱分析数据，在前端直接合成旋律音频并播放。
2. 播放过程中，同步高亮当前正在播放的简谱音符。
3. 用户可以调整播放速度，且调整后无需重新请求后端。
4. 本次为**纯前端改动**，不改后端接口、不改数据库、不改 Nginx。

## 4.2 模块二：播放器全局状态

### 目标
集中管理 Tone.js 实例、当前分析数据、调度事件、播放状态。

### 建议新增变量

```js
let toneReady = false;
let synth = null;
let scheduledIds = [];
let playbackEvents = [];
let currentAnalysis = null;
let originalBpm = 120;
let totalDurationBeats = 0;
let progressRaf = 0;
let currentHighlightEl = null;
```

### 说明
- `toneReady`：避免重复执行 `Tone.start()`。
- `synth`：Tone 合成器实例。
- `scheduledIds`：记录调度器 ID，停止时统一清理。
- `currentAnalysis`：缓存当前页面已加载的分析结果。
- `progressRaf`：用于驱动进度刷新。
- `currentHighlightEl`：追踪当前高亮元素，避免多个音符残留高亮。

---

## 4.3 模块三：音高重建函数

### 目标
根据简谱字段重建 MIDI / 频率。

### 建议新增函数

#### 1）`jianpuToMidi(note, keyInfo)`
作用：
- 读取 `note.degree`
- 根据 `keyInfo.mode` 选择大调/小调映射
- 根据 `keyInfo.tonic` 算出主音 pitch class
- 结合 `octave_offset` 推导最终 MIDI
- 对休止符返回 `null`

#### 2）`midiToFreq(midi)`
作用：
- 把 MIDI 号转为频率，供 Tone.js 播放

### 处理规则
- `degree === '0'` 或 `'-'`：按休止符处理
- 非法或未知 degree：按休止符处理
- `octave_offset` 缺失时按 0 处理

---

## 4.4 模块四：Tone.js 初始化

### 目标
只在用户点击播放后初始化音频上下文，规避浏览器自动播放限制。

### 建议新增函数

#### `ensureTone()`
职责：
1. 调用 `Tone.start()`
2. 初始化 `Tone.PolySynth`
3. 设置基础音色参数
4. 连接到 `destination`
5. 设置默认音量

### 音色建议
- 合成器：`Tone.PolySynth(Tone.Synth, ...)`
- 波形：`triangle`
- 原因：
  - 比正弦波更有存在感；
  - 比 saw/square 更柔和；
  - 适合简谱试听场景。

---

## 4.5 模块五：播放事件构建与调度

### 目标
把 `notation.notes` 转换为可调度的播放事件，并与 DOM 高亮建立映射。

### 核心设计
采用 **基于 beat 的调度**，而不是基于秒的调度。

这样做的优势：
1. 用户修改 BPM 后，Tone Transport 可自动整体变速；
2. 不需要在每次调速时重建所有事件；
3. 与后端 `duration_beats` 字段天然契合。

### 建议新增函数

#### 1）`beatsToBBS(totalBeats, numerator)`
作用：
- 将拍数转换成 Tone Transport 需要的 `Bars:Beats:Sixteenths` 格式

#### 2）`buildScheduleAndPlay(analysis)`
职责：
- 读取 `analysis.notation.notes`
- 计算每个音符的：
  - `startBeat`
  - `durBeat`
  - `midi`
  - 对应 DOM 元素
- 清除旧调度
- 为每个音符注册 Tone 调度事件
- 在调度回调里：
  - 播放音符
  - 高亮当前 DOM 节点
- 计算整段旋律总时长
- 在结尾注册自动停止逻辑

### DOM 映射要求
为了让播放事件能精确定位到当前页面中的音符节点，需要修改 `renderDegree()` 输出：
- 为简谱 DOM 增加 `data-note-index`

例如：
```html
<span class="degree" data-note-index="12">5</span>
```

此字段必须来自后端 `note.index`。

## 4.6 模块六：高亮控制

### 目标
播放过程中只让当前音符高亮，切换时自动移除上一个高亮。

### 建议新增函数

#### 1）`highlightEl(el)`
作用：
- 移除前一个高亮
- 给当前元素加 `.playing`

#### 2）`clearHighlight()`
作用：
- 停止播放、切页、重播时清除残留高亮

---

## 4.7 模块七：播放控制行为

### 目标
实现播放 / 暂停 / 继续 / 停止。

### 建议新增函数

#### 1）`togglePlay()`
职责：
- 首次播放时初始化 Tone
- 如果当前在播放：执行暂停
- 如果当前是停止状态：构建调度并从头播放
- 如果当前是暂停状态：从当前位置继续
- 更新按钮文案和禁用状态
- 开启进度刷新

#### 2）`stopPlayback()`
职责：
- 停止 Tone Transport
- 清空调度事件
- 清空高亮
- 重置进度显示
- 恢复按钮初始状态

#### 3）`onTempoChange(value)`
职责：
- 更新界面上的 BPM 文案
- 若 Tone 已初始化，则直接更新 `Tone.Transport.bpm.value`

### 特别说明
速度调整应支持：
- 播放前调整
- 播放中调整
- 暂停状态调整

由于采用 beat-based 调度，播放中调速无需重建调度。

---

## 4.8 模块八：进度显示

### 目标
实时展示当前播放进度和总时长。

### 建议新增函数

#### 1）`formatTime(sec)`
作用：
- 秒数格式化为 `m:ss`

#### 2）`updateProgressDisplay(elapsedBeats)`
作用：
- 根据当前 BPM 计算已播放秒数与总秒数
- 更新 `#progressText`

#### 3）`tickProgress()`
作用：
- 基于 `requestAnimationFrame` 持续刷新进度
- 当 Transport 停止时自动终止刷新

## 4.9 模块九：页面生命周期清理

## 8. 边界情况与处理策略

## 8.1 休止符
- 条件：`degree` 为 `0` 或 `-`
- 处理：不发声，但保留时间轴推进
- UI：可选择高亮该休止符，也可不高亮；建议仍按当前事件推进高亮逻辑，以保持视觉同步

## 8.2 未知音级
- 条件：degree 不在映射表中
- 处理：按休止符处理，不报错中断整体播放

## 8.3 空简谱数据
- 条件：`notation.notes` 和 `notation.measures` 都为空
- 处理：
  - 保持现有空态提示
  - 隐藏 `playerCard`

## 8.4 首次点击无声
- 原因：浏览器需要用户手势激活音频上下文
- 处理：只在 `togglePlay()` 中调用 `Tone.start()`，避免自动初始化

## 8.5 非 4/4 拍号
- 处理：读取 `analysis.time_signature` 设置 `Tone.Transport.timeSignature`
- 调度位置换算函数 `beatsToBBS()` 必须使用 `numerator`

## 8.6 播放中修改速度
- 处理：直接更新 `Tone.Transport.bpm.value`
- 不重建调度
- 进度文本根据当前 BPM 实时换算显示

## 8.7 切换歌曲 / 页面刷新
- 处理：统一调用停止与清理逻辑
- 防止 Transport 和高亮状态残留

## 9. 风险点与注意事项

## 9.1 Tone.js 时值换算要统一
风险：如果调度用 beat，而发声时值却按旧秒数写死，调速后可能出现音长不一致。

建议：
- 统一以 beat 为主数据源；
- 音符发声时长换算要基于当前 Transport BPM；
- 实现阶段需重点验证调速后的音长是否与视觉进度一致。

## 9.2 `measure.degrees` 与 `measureNotes[index]` 的对应关系依赖后端输出顺序
当前页面在 `renderMeasureCard()` 中通过数组下标把 `degrees` 与 `notesInMeasure()` 对齐。

风险：
- 如果未来后端 measure 中 degree 列表顺序与 notes 顺序不一致，当前视觉和播放定位可能错位。

当前判断：
- 现有后端 `_build_measures()` 是按 `notation_notes` 顺序推进，当前可用。

建议：
- 本次按现有结构执行；
- 实现阶段如发现个别小节存在错位，再评估是否改成以 `measure.notes` 明确索引渲染。

## 9.3 休止符没有 note.index
如果某些视觉上的“0”来自纯度数字符串而不是 notes 数据，播放高亮时无法精确绑定。

当前页面现状：
- `renderDegree()` 对休止符可单独渲染；
- 若无对应 note，允许没有 `data-note-index`。

建议：
- 高亮逻辑以真实 `notation.notes[]` 为主，不强行对所有视觉 rest 做绑定。

## 10. 验收清单

以下为可执行的前端验收步骤。

## 10.1 基础显示验收
1. 打开 `melody.html?song_id=<有效ID>`。
2. 页面正常显示调、拍号、BPM、置信度。
3. 当存在简谱数据时，播放器卡片显示。
4. 播放器卡片内出现：
   - 播放按钮
   - 停止按钮
   - 进度文本
   - 速度滑块

## 10.2 播放功能验收
1. 点击 `▶ 播放`。
2. 首次点击后，浏览器成功启用音频上下文。
3. 能听到按顺序播放的旋律音符。
4. 播放按钮文案切换为 `⏸ 暂停`。
5. 停止按钮变为可点击。

## 10.3 高亮同步验收
1. 播放中当前音符出现黄色高亮。
2. 随着播放推进，高亮逐个切换。
3. 同一时刻只保留一个当前高亮元素。
4. 停止后高亮全部清除。

## 10.4 暂停/继续验收
1. 播放中点击 `⏸ 暂停`。
2. 音频立即暂停。
3. 进度显示停止更新。
4. 再次点击后从当前位置继续播放，而不是从头开始。

## 10.5 停止验收
1. 播放中点击 `■ 停止`。
2. 音频停止。
3. Transport 重置到起点。
4. 进度显示恢复到 `0:00 / 总时长` 或初始值。
5. 播放按钮恢复 `▶ 播放`。
6. 停止按钮恢复禁用。

## 10.6 调速验收
1. 拖动速度滑块，右侧 BPM 数值实时变化。
2. 未播放前调整速度，点击播放后按新速度播放。
3. 播放中调整速度，旋律立即整体变快或变慢。
4. 调速后高亮切换节奏与声音保持同步。

## 10.7 调式验收
1. 使用一首大调歌曲测试：确认 1、3、5 等音级关系正确。
2. 使用一首小调歌曲测试：确认 `b3`、`b6`、`b7` 音高符合预期。
3. 对比页面显示的调性文本与实际听感，确认无明显错位。

## 10.8 边界验收
1. 对空简谱歌曲：页面显示空态，播放器不显示。
2. 对不存在的 `song_id`：维持现有错误提示逻辑，不显示播放器。
3. 页面刷新或跳转离开后，不应继续有残留音频。
