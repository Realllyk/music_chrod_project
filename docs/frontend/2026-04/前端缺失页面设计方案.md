## 1. 设计目标

根据现有前端页面、需求清单与 API 需求文档，补齐 7 个待建/待补全页面的页面结构、接口依赖、交互逻辑与验收标准，供后续前后端对齐与实现评审使用。

## 2. 统一设计约定

### 2.1 页面风格
- 延续现有前端样式：白底卡片、圆角 `12px`、主色 `#4ecdc4`
- 保留统一菜单：`<script src="/js/menu.js"></script>`
- 所有请求统一使用：`const API_BASE = ''`
- 页面跳转统一使用：`window.location.href = '...'`
- 异常提示延续现有方式：`alert()`

### 2.2 通用异常处理约定
| 场景 | 前端处理 |
|------|----------|
| URL 缺少 id / song_id | `alert('参数缺失')` 后跳回列表页 |
| 接口 404 | 提示“数据不存在或已删除”，返回上一页或列表页 |
| 接口 500 | 提示“服务异常，请稍后重试” |
| 网络异常 | 提示“网络请求失败” |
| 删除操作 | 二次确认 `confirm()` |
| 空数据列表 | 展示空状态文案，不展示空表格 |

### 2.3 待后端确认项
以下字段先按假设设计，文档中均标注为 **待后端确认**：
1. `GET /api/songs/<song_id>` 是否返回：`melody_path`、`chord_path`
2. `GET /api/transcribe/song/<song_id>` 的返回字段名称与结果路径字段命名
3. `GET /api/audio-sources/<audio_id>` 返回的音频预览字段是 `file_path` 还是 `audio_url`
4. `GET /api/capture/detail/<session_id>` 返回字段中音频访问路径字段命名
5. `GET /api/status` 是否直接提供首页统计数据；若无，则首页退化为仅展示快捷入口

### 4.1.2 页面结构

#### 页面结构说明
- 返回按钮
- 标题区：歌曲名称 / 状态
- 基本信息卡片
- 音源信息卡片
- **新增：转写结果卡片**
- **新增：快捷操作卡片**
- 原有操作卡片（保存 / 删除）

#### 线框图
```text
┌─────────────────────────────────────────────┐
│ ← 返回列表                                  │
│ 🎵 歌曲详情                                  │
├─────────────────────────────────────────────┤
│ [基本信息卡片]                               │
│ ID | 歌曲名 | 歌手 | 分类                     │
├─────────────────────────────────────────────┤
│ [音源信息卡片]                               │
│ 音源路径 | 音频播放器 | 状态                  │
├─────────────────────────────────────────────┤
│ [转写结果卡片]                               │
│ 旋律 MIDI： [下载] / 未生成                   │
│ 和弦 MIDI： [下载] / 未生成                   │
│ 最近转写时间：xxxx（可选）                    │
├─────────────────────────────────────────────┤
│ [快捷操作卡片]                               │
│ [快速提取旋律] [快速提取和弦] [查看转写历史]   │
├─────────────────────────────────────────────┤
│ [操作卡片]                                   │
│ [保存修改] [删除]                            │
└─────────────────────────────────────────────┘
```

### 4.1.3 API 调用表

| API | 方法 | 用途 | 调用时机 |
|-----|------|------|----------|
| `/api/songs/{id}` | GET | 加载歌曲详情 | 页面初始化 |
| `/api/artists/list` | GET | 加载歌手下拉 | 页面初始化 |
| `/api/transcribe/start` | POST | 快速发起提取 | 点击快捷提取按钮 |
| `/api/songs/{id}` | PUT | 保存编辑 | 点击保存 |
| `/api/songs/{id}` | DELETE | 删除歌曲 | 点击删除 |

### 4.1.4 API 详细说明

#### 1) 获取歌曲详情
- **路径**：`GET /api/songs/{id}`
- **请求参数**：

| 参数 | 位置 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| id | path | number | 是 | 歌曲 ID |

- **假设响应字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | number | 歌曲 ID |
| title | string | 歌曲名 |
| artist_id | number/null | 歌手 ID |
| artist_name | string | 歌手名 |
| category | string | 分类 |
| audio_path | string | 音源路径 |
| status | string | 歌曲状态 |
| melody_path | string/null | 旋律 MIDI 路径，**待后端确认** |
| chord_path | string/null | 和弦 MIDI 路径，**待后端确认** |
| updated_at | string | 更新时间 |

- **异常处理**：
  - 404：提示歌曲不存在，跳回 `/songs`
  - `melody_path/chord_path` 缺失：按“未生成”展示，不报错

#### 2) 发起转写任务
- **路径**：`POST /api/transcribe/start`
- **请求体**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| song_id | number | 是 | 歌曲 ID |
| mode | string | 是 | `melody` / `chord` |

- **响应字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| ok | boolean | 是否成功 |
| task_id | string/number | 任务 ID |
| error | string | 失败原因 |

- **异常处理**：
  - `ok=false`：提示提交失败
  - 任务创建成功后：提示“任务已提交”，可选择跳转 `/transcribe/history?song_id={id}`

#### 3) 保存歌曲
- **路径**：`PUT /api/songs/{id}`
- **请求体**：`{ title, artist_id, category }`
- **响应字段**：`{ ok, error? }`
- **异常处理**：保存失败时停留当前页并提示

#### 4) 删除歌曲
- **路径**：`DELETE /api/songs/{id}`
- **响应字段**：`{ ok, error? }`
- **异常处理**：删除成功后返回 `/songs`

### 4.1.5 交互逻辑
1. 页面读取 query `id`
2. 调用歌曲详情接口填充现有表单
3. 若 `melody_path` / `chord_path` 有值，则显示下载链接；否则显示“未生成”
4. 点击“快速提取旋律/和弦”
   - 按钮进入 loading
   - 调用 `/api/transcribe/start`
   - 成功后提示任务 ID，并提供“去查看历史”跳转
5. 点击“查看转写历史”跳转到 `/transcribe/history?song_id={id}`

### 4.1.6 验收清单
- [ ] 页面能正常展示歌曲基本信息
- [ ] 页面能展示音频播放器
- [ ] 转写结果卡片能展示旋律/和弦下载状态
- [ ] 点击“快速提取旋律”能成功提交任务
- [ ] 点击“快速提取和弦”能成功提交任务
- [ ] 点击“查看转写历史”跳转正确
- [ ] 保存、删除功能不受新增内容影响
- [ ] `melody_path/chord_path` 为空时页面不报错

## 4.2 音源列表页补全 `pages/audio-sources/index.html`

### 4.2.1 页面目标
在现有列表页中补齐详情/删除操作，并增加文件大小、上传时间等辅助信息展示。

#### 线框图
```text
┌────────────────────────────────────────────────────────────┐
│ 🎧 音源管理                                                │

├────────────────────────────────────────────────────────────┤
│ 名称 | 格式 | 大小 | 上传时间 | 状态 | 操作                │
│ xxx  | mp3  | 5MB  | xxxx     | active | [详情] [删除]     │
│ xxx  | wav  | 9MB  | xxxx     | active | [详情] [删除]     │
└────────────────────────────────────────────────────────────┘
```

### 4.2.3 API 调用表

| API | 方法 | 用途 | 调用时机 |
|-----|------|------|----------|
| `/api/audio-sources/list` | GET | 加载音源列表 | 页面初始化 / 刷新 |
| `/api/audio-sources/{id}` | DELETE | 删除音源 | 点击删除 |

### 4.2.4 API 详细说明

#### 1) 获取音源列表
- **路径**：`GET /api/audio-sources/list`
- **查询参数**：可选 `status=active`
- **假设响应字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| sources | array | 列表 |
| id | number | 音源 ID |
| audio_name | string | 音源名称 |
| format | string | 文件格式 |
| file_size | number/string | 文件大小 |
| created_at | string | 上传时间 |
| status | string | 状态 |
| file_path | string | 文件路径 |

- **异常处理**：
  - 空列表显示“暂无音源”
  - 某些字段缺失时用 `-` 占位

#### 2) 删除音源
- **路径**：`DELETE /api/audio-sources/{id}`
- **响应字段**：`{ ok, error? }`
- **异常处理**：
  - 删除前 `confirm('确定删除该音源？')`
  - 删除成功后刷新当前列表
  - 删除失败提示错误

### 4.2.5 交互逻辑
1. 页面初始化加载 `/api/audio-sources/list`
2. 每条数据渲染为一行，操作区包含：
   - `详情`：跳转 `/audio-sources/details?id={id}`
   - `删除`：调用删除接口
3. 删除成功后保留当前页并重新加载列表

### 4.2.6 验收清单
- [ ] 列表能展示音源名称、格式、大小、上传时间、状态
- [ ] 每行都有“详情”按钮
- [ ] 每行都有“删除”按钮
- [ ] 删除前有确认弹窗
- [ ] 删除成功后列表自动刷新
- [ ] 空列表状态展示正常

### 4.4.3 API 调用表

| API | 方法 | 用途 | 调用时机 |
|-----|------|------|----------|
| `/api/transcribe/song/{song_id}` | GET | 查询指定歌曲的转写历史 | 页面初始化 / 刷新 |
| `/api/songs/{song_id}` | GET | 获取歌曲标题（推荐） | 页面初始化 |

### 4.4.4 API 详细说明

#### 1) 获取歌曲信息（推荐）
- **路径**：`GET /api/songs/{song_id}`
- **用途**：显示顶部“歌曲名称”
- **异常处理**：若失败，标题退化为 `song_id={id}`

#### 2) 获取转写历史
- **路径**：`GET /api/transcribe/song/{song_id}`
- **请求参数**：路径参数 `song_id`
- **假设响应结构**：

| 字段 | 类型 | 说明 |
|------|------|------|
| tasks | array | 任务列表，**待后端确认字段名** |
| task_id | string/number | 任务 ID |
| mode | string | `melody` / `chord` / `polyphonic`，**待后端确认** |
| status | string | `queued/running/completed/failed` |
| created_at | string | 创建时间 |
| finished_at | string | 完成时间 |
| result_path | string | 结果文件路径，**待后端确认字段名** |
| error | string | 错误信息 |

- **异常处理**：
  - 接口返回空数组：展示“暂无转写历史”
  - 状态为 `failed`：展示错误信息入口
  - `result_path` 缺失：显示“无结果文件”

### 4.4.5 交互逻辑
1. 从 query 中读取 `song_id`
2. 并行加载歌曲标题与转写历史
3. 历史列表按创建时间倒序展示
4. `status=completed` 且有结果路径时，显示“下载”链接
5. 点击“返回歌曲详情”跳回 `/songs/details?id={song_id}`

- **响应字段**：`{ ok, error? }`
- **异常处理**：成功后返回 `/recordings`

### 4.5.5 交互逻辑

4. 删除时二次确认，成功后跳回录音列表
