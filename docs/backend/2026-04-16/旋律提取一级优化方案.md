## 阶段 4：数据交互说明

### 4.1 当前链路

```text
原曲
  ↓
demucs 分离人声
  ↓
vocal stem
  ↓
librosa PYIN 提取 pitch
  ↓
pitch 转 notes
  ↓
生成 MIDI
```

### 4.2 一级优化后的链路

```text
原曲
  ↓
demucs 分离人声（输出改为 WAV）
  ↓
vocal stem（信号质量更高）
  ↓
librosa PYIN 提取 pitch（读取配置中的 fmin/fmax）
  ↓
按 confidence_threshold 过滤低可信帧
  ↓
转换 notes
  ↓
按 min_note_duration 过滤极短假音符
  ↓
生成 MIDI
```

### 4.3 一级优化的核心收益

#### 修复 1：扩大音域覆盖
避免女声 / 高音旋律因为上限过低被直接截断。

#### 修复 2：过滤低置信帧
减少 demucs 残余底噪对旋律识别的污染。

#### 修复 3：过滤极短伪音符
减少毛刺和噪声造成的碎片音符。

#### 修复 4：demucs 输出 WAV
避免 MP3 再压缩对 pitch 检测精度造成二次伤害。

## 一级优化实施内容

### 1. melody 参数下沉到 config.json
建议在 `transcription.melody` 中增加或明确以下配置：

```json
"melody": {
  "fmin": 65,
  "fmax": 1047,
  "hop_length": 512,
  "window_size": 7,
  "confidence_threshold": 0.5,
  "min_note_duration": 0.08
}
```

说明：
- `fmin=65`：约等于 C2
- `fmax=1047`：约等于 C6，覆盖绝大多数流行人声
- `confidence_threshold=0.5`：过滤低可信帧
- `min_note_duration=0.08`：过滤 80ms 以下极短假音符

### 2. librosa melody 读取配置并应用
需要修改 `backend/transcriber/librosa/melody.py`：
- `extract_pitch_pyin()` 中读取 `fmin/fmax`
- `pitch_to_notes()` 中读取 `confidence_threshold`
- 音符生成完成后按 `min_note_duration` 过滤

### 3. demucs 输出格式改为 WAV
修改 `backend/config.json`：

```json
"demucs": {
  "output_format": "wav"
}
```

说明：
- demucs 输出 WAV 后，给 librosa 的输入信号质量更高
- `backend/transcriber/demucs/melody.py` 已经优先查找 `vocals.wav`，本项主要是调整配置

### 4. 建议补充的关键日志
建议在 `librosa/melody.py` 中增加少量关键日志：
- 当前读取到的 `fmin/fmax`
- `confidence_threshold`
- 原始音符数
- 过滤后的音符数

这样便于判断一级优化是否真正起效。
