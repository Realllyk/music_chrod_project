# 接口规范化 - 阶段 C2 后端实现报告

## 1. 实现目标

按已批准文档《接口规范化-阶段C2-capture接口设计》完成 `backend/controllers/capture_controller.py` 规范化改造：

- capture 相关接口统一改为 DTO 入参
- JSON 接口统一使用 `Result.success(...).to_response()`

## 5. Controller 改造结果

已完成以下接口规范化：

### Session 管理

- `POST /api/capture/start`
- `GET /api/capture/active`
- `PUT /api/capture/stop`
- `GET /api/capture/list`
- `GET /api/capture/detail/<session_id>`
- `DELETE /api/capture/sessions/<session_id>`
- `PUT /api/capture/sessions/<session_id>`

### Recording 控制

- `PUT /api/capture/request-recording`
- `POST /api/capture/start-recording`
- `PUT /api/capture/stop-recording`
- `PUT /api/capture/save`
- `PUT /api/capture/register-file`
- `POST /api/capture/upload-file`

### 录音回放

- `GET /api/capture/recordings`
- `GET /api/capture/uploads/recordings/<filename>`

改造后的 controller 特征：

- 不再使用 `request.get_json()`
- 不再使用 `request.form.get()`
- 不再使用 `request.args.get()`
- 不再返回 `jsonify({'error': ...})`
- JSON 接口统一 `Result.success(...).to_response()` / `Result.not_found(...).to_response()` / `Result.bad_request(...).to_response()` 等
