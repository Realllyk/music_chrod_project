## 3. 关键实现说明

### 3.1 DTO / VO 落地

新增 DTO：
- `ListArtistsQueryDTO`
- `ArtistIdPathDTO`
- `AddArtistDTO`
- `UpdateArtistDTO`
- `AvatarUploadDTO`

新增 VO：
- `ArtistVO`

其中：
- `AddArtistDTO` 继续通过 `@use_dto(AddArtistDTO)` 保持 JSON / multipart 双模式兼容
- `UpdateArtistDTO` 通过 Pydantic 校验确保至少提供 `name` 或 `bio`
- `AvatarUploadDTO` 对头像文件缺失场景做 DTO 层校验
- `ArtistVO` 统一输出 `id/name/avatar/bio/created_at/updated_at`

### 3.2 controller 规范化

`backend/controllers/artists_controller.py` 已完成以下收敛：
- 不再直接使用 `request.args / request.form / request.files / request.get_json / request.json`
- 不再使用 `jsonify({'error': ...})` 或其他裸错误响应
- 全部接口统一返回 `Result.success(...).to_response()`
- 参数错误 / 资源不存在 / 冲突错误通过异常体系交给全局 error handler 处理

### 3.3 service 语义补齐

对 `backend/services/artists_service.py` 做了最小必要改造：
- `name` 缺失 → `BadRequestException`
- 歌手不存在 → `NotFoundException`
- 歌手名重复 / 删除前被歌曲引用 → `ConflictException`
- 数据库写入失败保留为运行时异常，由全局兜底返回 500

### 3.4 avatar 返回字段修正

头像专用接口已统一返回：

```json
{
  "ok": true,
  "avatar_url": "..."
}
```

已修正旧字段名 `avatar`，满足验收要求。
