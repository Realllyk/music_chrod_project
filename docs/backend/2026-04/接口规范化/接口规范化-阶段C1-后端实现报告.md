# 接口规范化 - 阶段 C1 后端实现报告

## 1. 实现目标

按已批准方案，对 `backend/controllers/transcribe_controller.py` 的 3 个接口完成 DTO / VO / Result 规范化改造：

1. `POST /api/transcribe/start`
2. `GET /api/transcribe/status/<task_id>`
3. `GET /api/transcribe/song/<song_id>`

同时新增专用 DTO、VO 文件，并将 controller 内的任务序列化逻辑迁移到 VO 层。

---

## 2. 本次改动清单

### 2.1 新增 DTO

文件：`backend/pojo/dto/transcribe_dto.py`

已新增：
- `StartTranscribeDTO`
- `TranscribeTaskQueryDTO`
- `SongTasksQueryDTO`

字段实现：
- `StartTranscribeDTO.song_id: int(gt=0, required)`
- `StartTranscribeDTO.mode: Literal['melody', 'chord'] = 'melody'`
- `TranscribeTaskQueryDTO.task_id: str(min_length=1, required)`
- `SongTasksQueryDTO.song_id: int(gt=0, required)`

### 2.2 新增 VO

文件：`backend/pojo/vo/transcribe_vo.py`

已新增：
- `StartTranscribeVO`
- `TranscribeTaskVO`
- `SongTranscribeTasksVO`

其中：
- `StartTranscribeVO` 返回 `task_id`、`status`
- `TranscribeTaskVO` 统一承载单任务响应字段
- `SongTranscribeTasksVO` 返回 `song_id + tasks`

### 2.3 Controller 改造

文件：`backend/controllers/transcribe_controller.py`

已完成：
- 3 个接口全部加上 `@use_dto`
- 函数签名改为 DTO 入参
- 删除 controller 中直接使用 `request.get_json()` 的写法
- 删除 controller 中 `jsonify({'error': ...})` / 裸结构返回
- 成功响应统一改为 `Result.success(vo).to_response()`
- 原 `_serialize_task(task)` 逻辑迁移到 `TranscribeTaskVO.from_domain(task)`

### 2.4 错误处理改造

已完成：
- 歌曲不存在：抛 `NotFoundException('Song not found')`
- 任务不存在：抛 `NotFoundException('Task not found')`
- 参数非法不再手写返回裸错误 JSON，由 DTO + 全局错误处理统一接管

---

## 3. 接口实现结果

### 3.1 `POST /api/transcribe/start`

实现形态：
- `@use_dto(StartTranscribeDTO)`
- 成功返回：`Result.success(StartTranscribeVO(task_id=..., status='pending')).to_response()`

当前行为：
- DTO 自动校验 `song_id`、`mode`
- 补充歌曲存在性校验，不存在时返回 404 业务异常
- 任务创建成功后启动后台线程执行转写

### 3.2 `GET /api/transcribe/status/<task_id>`

实现形态：
- `@use_dto(TranscribeTaskQueryDTO, source='path')`
- 成功返回：`Result.success(TranscribeTaskVO.from_domain(task)).to_response()`

当前行为：
- 路径参数进入 DTO
- 不存在任务时抛 `NotFoundException`
- 单任务返回结构已统一到 `Result.result`

### 3.3 `GET /api/transcribe/song/<song_id>`

实现形态：
- `@use_dto(SongTasksQueryDTO, source='path')`
- 成功返回：`Result.success(SongTranscribeTasksVO(...)).to_response()`

当前行为：
- 路径参数进入 DTO
- 返回结构为：
  - `result.song_id`
  - `result.tasks`
- 每个任务元素均使用 `TranscribeTaskVO`

## 5. 实现说明与边界

### 5.1 `progress`

当前任务表与原有后台处理逻辑未提供精确进度落库能力，因此：
- `TranscribeTaskVO.progress` 已按设计保留
- 当前返回值默认从任务数据中读取该字段
- 若数据库暂无该字段，则返回 `null`

### 5.2 `vocal_stem_path`

VO 层已保留 `vocal_stem_path` 字段并支持从领域对象映射。
当前 controller 不再手写条件拼装字段。

### 5.3 非业务异常

数据库连接失败、任务创建失败等基础设施问题，目前统一抛出运行时异常，由全局异常处理转换为标准 `Result` 错误响应。
