## 2. 关键实现说明

### 2.1 调性估计
- 统计所有音符的时长加权 pitch-class 直方图
- 与 24 个调（12 大调 + 12 小调）的 Krumhansl profile 做相关性比对
- 输出：
  - `tonic`
  - `mode`
  - `display`
  - `confidence`

### 2.2 简谱映射
- 以估计调的 tonic 作为参考
- 将 `midi % 12` 映射到：
  - 大调：`1 2 3 4 5 6 7` + `#` 变化音
  - 小调：包含 `b3 / b6 / b7`
- 输出每个音的：
  - `pitch_name`
  - `degree`
  - `octave_offset`
  - `duration_beats`

### 2.3 小节分组
- 优先使用 MIDI 自带拍号
- 无拍号时默认 `4/4`
- 结合 tempo 计算小节时长
- 按 `start / measure_duration` 分桶生成 `notation.measures`

### 2.4 缓存策略
- `song_analysis` 复用为分析缓存表
- 通过 `song_id + analysis_type` 保证 melody 结果可覆盖更新
- 通过 `analysis_key` 与 `songs.melody_key` 绑定当前生效缓存

如需上线前人工执行一次 SQL 变更，可使用以下语句：

```sql
ALTER TABLE songs
  ADD COLUMN melody_key VARCHAR(32) DEFAULT NULL COMMENT '旋律分析缓存key' AFTER chord_path,
  ADD COLUMN chord_key VARCHAR(32) DEFAULT NULL COMMENT '和弦分析缓存key' AFTER melody_key;

ALTER TABLE song_analysis
  ADD COLUMN analysis_key VARCHAR(32) DEFAULT NULL COMMENT '分析记录唯一业务key' AFTER analysis_type,
  ADD COLUMN updated_at TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间' AFTER created_at,
  ADD UNIQUE KEY uk_analysis_key (analysis_key),
  ADD INDEX idx_song_type (song_id, analysis_type);
```
